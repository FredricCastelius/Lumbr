Files

This is the including package. To keep things simple we export and minify all style and script files and put them in the root. Making the page load faster with fewer page requests.

The two important files to keep in mind is settings.scss and layout.scss

Settings

In the _settings.scss file you define your project with all its basic yet very useful settings like typography heriarki, colors on all links, hover animation speed just to name a few.

Layout

In the _layout.scss file is were you will put in your own layout starting fresch with just your code. Lumbr is just there in the background.

Settings

When fireing up a new project the first thing you should do is to get in to the _settings.scss. Here is all the main settings for the project. Later you will add your own layout in _layout.scss but first, lets take a look at the settings file:

  

// desktops
$desktopWidth: 1200px; // when desktops starts
$desktopLayoutBase: 120%; // general font size and layout spacing

// laptops
$laptopWidth: 960px; // when laptops starts
$laptopFontBaseSize: 130%; // general font size and layout spacing

// tablets
$tabletWidth: 768px ; // when tablet starts
$tabletFontBaseSize: 120%; // general font size and layout spacing

// phones
$phoneFontBaseSize: 120%; // general font size and layout spacing

// layout max point
$layoutMaxPoint: 1100px;

Define breakpoints

The $desktopWidth, $laptopWidth and the $tabletWidth define your projects breakpoints. We know that devices and breakpoints differ from different projects so we made it easy for you to change.

The BaseSize is a very interesting number. Let's say it's our magical number. It represents which base the elastic layout should calculate it's layout from. This is achived by using Rems in the styling.

This really comes in handy when you want bigger text and consistent layout on different devices.


// base type
$font-weight: 300;
$line-height: 1.65;
$font-size: 80; // percentage value (16 * 112.5% = 18px)
$font-base: 16 * ($font-size/100); // converts our percentage to a pixel value

Layout

When your settings file is set up after you projects preferences lets head over to layout.scss. As you see its a empty canvas for you to add your awesomly creafted style. For rapid up your styling processes we have created and tweakded some of the newest and coolest featuers for you to use easily.

Elastic spacing

For creating that elastic effect we need elastic spacing and that we have made a function for. It's called fnGetSpacing().The value you put in, in the function is then based on the magicnumber from the settings file!


div {
    padding: fnGetSpacing(100); // elastic spacing
}

Grids

We all need those grids. Havn't that been alittle standardized latley? lets shake this up with the property flexbox and mixins.


#wrapper {
    @include wrapper();
    aside {
        width: 200px; // elastic spacing
        margin: 0 fnGetSpacing(100) 0 0; // elastic spacing
    }
    article {
        flex: 1;
    } 
}

Trigger viewports

It should be easy to trigger a viewport, with Lumbr it's as easy as baking a pie, well no, simplier. Here's the snippets


standard ones
@include breakpoint(desktop){} Trigger the dekstop which is the highest viewport
@include breakpoint(laptop){} Triggers the laptop only
@include breakpoint(tablet){} Triggers the tablet only
@include breakpoint(phone){} Triggers the phone only

// special ones
@include breakpoint(laptopUp){} Triggers the laptop and up
@include breakpoint(laptopDown){} Triggers the laptop and down
@include breakpoint(tabletUp){} Triggers tablet and up
@include breakpoint(tabletDekstop){} Covers tablet to dekstop
